# RedCheck Protocol Generator

Настольное приложение с графическим интерфейсом для автоматического формирования протокола информационной безопасности в формате Word (DOCX) на основе отчетов RedCheck.

## Возможности

- **Парсинг XML отчетов RedCheck:**
  - Отчет об инвентаризации (хосты, ОС, ПО, пользователи, службы, обновления)
  - Отчет пентеста (открытые порты, сервисы, баннеры, SMB конфигурация)
  - Отчет сканирования уязвимостей (CVSS, критичность, эксплойты)

- **Генерация DOCX документов:**
  - Заполнение таблиц в шаблоне данными из отчетов
  - Поддержка русских и английских заголовков таблиц
  - Сохранение форматирования шаблона

- **Графический интерфейс (Tkinter):**
  - Выбор файлов отчетов
  - Выбор шаблона и выходного файла
  - Индикатор прогресса
  - Логирование в реальном времени

- **Устойчивость к ошибкам:**
  - Обработка неполных/поврежденных XML файлов
  - Логирование ошибок и предупреждений
  - Продолжение работы при частичных ошибках

## Установка

Требуемые зависимости:
```bash
pip install python-docx lxml pillow
```

## Использование

### Графический интерфейс

```bash
python main.py --gui
# или просто
python main.py
```

### Командная строка

```bash
python main.py \
    --inventory inventory_report.xml \
    --pentest pentest_report.xml \
    --vulnerability vuln_scan.xml \
    --template template.docx \
    --output protocol.docx \
    --verbose
```

### Краткие опции

```bash
python main.py -i inv.xml -p pentest.xml -v vulns.xml -t template.docx -o output.docx -v
```

## Модели данных

### Host (инвентаризация)
- IP адрес, hostname, ОС, архитектура, роль в домене
- Сетевые интерфейсы (IP, маска, шлюз, DNS, MAC)
- Пользователи и группы
- Службы и процессы
- Установленное ПО и обновления (KB)
- Политики безопасности

### PortScan (пентест)
- IP адрес, hostname
- Открытые порты (номер, протокол, состояние)
- Сервисы (название, продукт, версия)
- Баннеры
- SMB информация (подпись, домен, версии)

### Vulnerability (уязвимости)
- ID, CVE ID, название
- CVSS score, критичность
- Наличие эксплойта, статус эксплуатации
- Целевой IP и порт
- Описание и рекомендации по устранению

## Шаблон документа

Шаблон DOCX должен содержать таблицы с заголовками на русском или английском языке:

| Таблица | Заголовки (примеры) |
|---------|---------------------|
| Хосты | IP адрес, Hostname, ОС, Архитектура, Роль в домене, ПО |
| Порты | IP адрес, Hostname, Порт, Протокол, Сервис |
| SMB | IP адрес, Подпись требуется, Подпись включена, Домен, Версия OS |
| Уязвимости | ID, CVE, Название, CVSS, Критичность, Эксплойт, Описание |

Метаданные заполняются через плейсхолдеры: `{{scan_type}}`, `{{profile}}`, `{{start_time}}`, `{{end_time}}`

## Логирование

Логи сохраняются в директорию `logs/` с меткой времени в имени файла.
